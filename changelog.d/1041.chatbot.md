The events rollup now aggregates `extra_miles_earned` into a new `user_rollups.extra_miles` column, and a new admin `!givemiles <user> <amount>` command applies manual miles corrections (may be negative) as auditable `correction` events. Reconstructed display miles ≈ `events_miles + extra_miles`, where `extra_miles` sums the sub-grant, 5%-bonus, and manual-correction portions the login/logout pairing can't see — while `events_miles` stays the pure pairing base (its delta from `users.miles` is the drift alarm).
