**Removed tripbot's in-process YouTube auth + Data-API client.** YouTube OAuth and runtime now live entirely on the platform-gateway (`gateway-youtube`) — tripbot holds no YouTube token at any point. Deletes `pkg/youtube`, `cmd/youtube-chat-spike`, the YouTube legs of the `/auth` handlers, the in-process chat send/poll path, and the `YOUTUBE_CLIENT_ID` / `YOUTUBE_CLIENT_SECRET` / `YOUTUBE_CHANNEL_ID` config. `YOUTUBE_API_URL` is now required on a `PLATFORM=youtube` instance; without it the instance comes up with no YouTube chat (logged loudly), the same stay-up contract as a missing Twitch token.
