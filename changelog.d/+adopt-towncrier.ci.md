**Changelog entries now live as [towncrier](https://towncrier.readthedocs.io) fragments.** Each PR drops a file in `changelog.d/` (e.g. `889.fix.md`) instead of editing `CHANGELOG.md`'s `[Unreleased]` block; the fragments are collated into a release section by `task changelog:build VERSION=x.y.z` at cut time. A `changelog` CI check requires a fragment on every PR into `develop` (escape with the `skip-changelog` label), so the release changelog stays complete by construction — and parallel PRs no longer conflict on `CHANGELOG.md`. The `pending-release` PR previews the assembled notes via `towncrier build --draft`.
