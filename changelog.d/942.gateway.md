**`tripbot_gateway_up` reachability metric.** Tripbot now records a gauge on every platform-gateway call — 1 when the call gets an HTTP response (gateway reachable), 0 on a transport failure — at the two `http.Do` chokepoints in the gateway client, covering all v1 calls and the chat-send path. It's the consumer-side signal for a gateway down-alert: it catches "the bot can't reach the gateway" (network partition, gateway crash, DNS), which the gateway's own `platform_gateway_up` process-liveness gauge can't report.
